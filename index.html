Design Process

  • Identifying the Problem

    The beginning of it all, this is part of the research and discover phase wherein a problem statement is identified and acknowledged. From here therein, A problem itself is to be explored and scrutinized for the possible solution.

  • User Persona

    This is the part where we establish a role for that sole problem in which we are focused on solving. Adding a persona creates a communication where we show empathy between the designer and the user to fully understand the connection existing in a problem and a possible solution.

  • Wireframe + Prototype

    After establishing user discussion, it is now time to create possible solution in terms of low-fidelity sketch or adding user diagrams to visualize the usability of the solution. It may be done on paper or any medium that means to deliver the demonstration for the initial idea.

  • User Test Report

    This is the key document to prove the usefulness of the solution created. Conducting user test aims to dig deeper on users' needs and pain points to be able to discuss with the team and understand what's going on and how the design should solve the problem.
